What Data Stays Local

Document Organizer is designed with privacy as a core principle. The following data never leaves your computer:

  • Your documents - PDFs, images, Word files, etc. are processed entirely locally
  • Original files - Only moved/renamed on your local filesystem
  • Metadata files - The .meta.json sidecar files are stored locally
  • Settings - Stored in your user config directory
  • Hash index - Used for duplicate detection, stored locally

What May Be Sent Externally

Depending on your AI provider choice, some data may be sent to external services:

If You Use Cloud AI Providers (Claude, GPT, Bedrock)

‚ö† Extracted text only is sent to the AI provider for categorization.

This means if you have a scanned bank statement, the text content (account numbers, balances, transaction details) will be sent to Anthropic, OpenAI, or AWS for analysis. The actual PDF/image file is NOT sent - only the extracted text.

Please review the privacy policies of your chosen AI provider:

  • Anthropic Privacy Policy
  • OpenAI Privacy Policy
  • AWS Privacy Policy

If You Use Ollama (Local AI)

‚úì Nothing is sent externally. Ollama runs entirely on your machine. All AI processing happens locally. This is the recommended option for maximum privacy.

If You Use Keyword Mode

‚úì Nothing is sent externally. Simple pattern matching runs locally without any AI calls.

API Keys

If you provide API keys for cloud AI providers, these are stored locally in your user settings file:

  • macOS: ~/Library/Application Support/DocumentOrganizer/settings.json
  • Linux: ~/.config/DocumentOrganizer/settings.json
  • Windows: %APPDATA%/DocumentOrganizer/settings.json

‚ö† API keys are stored in plain text. Ensure your system has appropriate access controls.

This Website

This landing page (the GitHub Pages site you're viewing) does not:

  • Use cookies
  • Track visitors with analytics
  • Collect any personal information
  • Use any third-party scripts

The only external request made is checking if your local Streamlit server is running on localhost:8501, which stays entirely on your machine.
